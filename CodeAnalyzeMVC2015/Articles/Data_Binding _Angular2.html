Types of data binding in Angular 2 with example

  1. We have 4 types of data binding in Angular 2
    1. Interpolation
    2. One-way binding
    3. Two-way binding
    4. Event binding
    Lets take at each of these with example



  2. 1. Interpolation
    This is similar to Expressions in previous versions, here we will be passing property name in View Template within double curly braces. Take a look at the example below

    app.component.ts 
    import { Component } from '@angular/core';  
@Component({  
  selector: 'my-app',  
  templateUrl: './app/databinding.html'  
})  
export class AppComponent {   
    name = 'Angular 2 Interpolation';  
}
    On the View html page we will be using this as follows 
    <div>  
    <h5>Interpolation Angular 2</h5> 
    This is an example for {{name}} !  
</div>


  3. 2. One-way binding
    Angular 2 uses HTML DOM element property for one-way binding. one-way data binding directive is replaced with [property]. We will be using square brackets to bind the element in the view as shown below

    app.component.ts 
    import { Component } from '@angular/core';  
@Component({  
    selector: 'my-app',  
    templateUrl: './app/databinding.html'  
})  
export class AppComponent {   
    title = 'Angular 2 One way data binding';  
    dataString = 'CodeAnalyze'  
}
    On the View html page we will be using this as follows 
    <div>  
    <h5>One way binding Angular 2</h5>  
    Hello <span [innerText] = "title" ></span>!  
    <input type='text'  [value]="dataString" />  
</div>


  4. 3. Two-way binding
    Angular 2 uses [(ngModel)]. We need to import FormsModule to the template module before using the ngModel directive as shown below

    app.module.ts 
    import { NgModule }      from '@angular/core';  
import { BrowserModule } from '@angular/platform-browser';  
import { FormsModule } from '@angular/forms';  
  
import { AppComponent }  from './app.component';  
  
@NgModule({  
  imports:      [ BrowserModule, FormsModule],  
  declarations: [ AppComponent],  
  bootstrap:    [ AppComponent ]  
})  
export class AppModule {   
}
    On the View html page we will be using this as follows 
    <div>  
    <h5>Two way binding Angular 2</h5>  
    Enter UserName:  <input [(ngModel)]="enterUserName"  />  
    <br/><br/>  
    <span> WelCome {{enterUserName}} ! </span>  
</div>
    This can be done without using ngModel also To imlement this we will use something called as "template reference variables". With this we will have direct access to an element. This identifier is prefixed with a hash/pound character (#). "txtName" is template reference variable below pointing to input element. Template does not bind to the component. This solution will not work unless we bind to the element’s event. This data binding is updated on any of the user actions only like a keypress event for exmaple. So we bind keyup event to input element and we do nothing with this event. We bind 0 with the event as shown below. 
    <div<  
    <h5<Two way binding (without ngModel directive)</h5<  
    Enter you Name: <input  #txtName  type = "text"  (keyup)="0" /<  
    <br/<<br/<  
    <span< WelCome {{txtName.value}} ! </span<  
</div<



  5. 4. Event Binding
    Angular 2.0 directly uses the valid HTML DOM element events.ng-click is now replaced with (click). The round brackets (parentheses) are used with DOM event name for event binding in Angular 2.

    app.component.ts 
    import { Component } from '@angular/core';  
@Component({  
  selector: 'test-app',  
  templateUrl: './app/databinding.html'  
})  
export class AppComponent {   
    messageText = '';  
    onClickMe() {  
        this.messageText = "This is an example for Event Binding in Angular 2";  
    }  
}
    On the View html page we will be using this as follows 
    <div>  
    <h5>Event binding Example</h5>  
     <button (click)="onClickMe()">Click me!</button>  
    <br/><br/>  
    <span> {{messageText}} </span>  
</div>

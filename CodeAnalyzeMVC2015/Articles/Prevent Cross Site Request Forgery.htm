Cross Site Request Forgery Attacks is mainly the requests used by the hackers to enter into our website and perform harmful actions. Normally this happens with Session mixup. When we are logged into multiple websites in the web browser there are chances that session of one website mix up with the another website. So in order to recognize and identify the client correctly we need to use anti forgery token.

 

Cookies have to be enabled by the client else this method will not work because server uses cookie to identify the request.

 

Controller looks like this

 

        [ValidateAntiForgeryToken]

        public class AccountUpdate : Controller

        {

            public ActionResult DeleteTransaction()

            {

                var transactionId = (int)Session["transactionId"];

                DeleteTransaction(transactionId);

                return View();

            }

        }

 

View

 

                @using (Html.BeginForm("DeleteTransaction", "User"))

      { 

            @Html.AntiForgeryToken()

            <input type="submit" value="Transfer Amount" />

      }

 

 

If the request is not valid then below error is thrown

 

The required anti-forgery cookie "__RequestVerificationToken" is not present.

 

Description: An unhandled exception occurred during the execution of the current web request.

Please review the stack trace for more information about the error and where it originated in the code.

 

Exception Details: System.Web.Mvc.HttpAntiForgeryException:

The required anti-forgery cookie "__RequestVerificationToken" is not present.

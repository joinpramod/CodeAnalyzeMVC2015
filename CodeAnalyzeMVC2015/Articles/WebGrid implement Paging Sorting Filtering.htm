WebGrid has inbuilt features for most of the features we need or have in ASP.NET gridview

 

Sample webgrid code on view is as shown below

<div id="gridContent">

@{

        var gd = new WebGrid(Model, canPage: true, rowsPerPage: 5, canSort: true,  ajaxUpdateContainerId: "gridContent");

       

}

</div>

 

 

 

canPage               -attribute is for Paging

rowsPerPage       -attribute is for pagesize

canSort                 -attribute is for sorting

 

 

ajaxUpdateContainerId – attribute is for implementing ajax, this is wrapping the whole webgrid object in  div as shown below

@gd.GetHtml(tableStyle: "csstable",

                headerStyle: "csshead",

                alternatingRowStyle: "cssaltRow",

                selectedRowStyle: "cssselectRow",

                columns: gd.Columns(

                         gd.Column("FirstName", " FirstName"),

                         gd.Column("LastName", " LastName"),

                         gd.Column("Address", "Address", style: "description"),

                         gd.Column("Age", " Age"),

                         gd.Column("State", " State"),

                         gd.Column("UserName", "UserName"),

                         gd.Column("Details", format: @<text>@Html.ActionLink("Details", "Details", new { id = item.Id }, "") </text>),

                         gd.Column("Edit", format: @<text>@Html.ActionLink("Edit", "Edit", new { id = item.Id }, "") </text>))

    ) 

 

 

tableStylee                           -attribute is to assign css style class for whole grid

headerStyle                       - attribute is to assign css style class for header

alternatingRowStyle- attribute is to assign css style class for alternating roww

selectedRowStyle-attribute is to assign css style class for selected row

columnss                               - is to customize the column names, can add some button columns if needed   and so on
